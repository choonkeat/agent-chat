Yep — that's the export_chat_html MCP tool (added in be0b93f, evolved through e55202e). Here's how it works end-to-end:

Flow
  1. Agent calls the export_chat_html MCP tool with a title (kebab-case slug). Optional: target_path, image_mode, timeout_seconds.
  2. Server publishes an exportRequest event over the transient bus to every connected browser tab.
  3. The browser walks its own DOM (so what's exported matches what you actually see — markdown, code blocks, canvas, quick replies, etc.), inlines uploaded images as base64 data URIs, bundles JS/CSS, and POSTs the resulting self-contained HTML back.
  4. Server writes it to disk and returns the path + byte count.
Path defaults (tools.go:492-502)
  • No target_path → ./agent-chats/YYYY-MM-DD-{slug}.html, with auto-suffix -2, -3 on collision.
  • Explicit target_path is allowed but rejected if it escapes cwd (path-traversal guard at tools.go:485-491).
Image mode (tools.go:504-515)
  • fullsize (default): keeps original image bytes; thumbnails are clickable and open in a lightbox (the e55202e fix — they used to open in a new tab).
  • thumbnail: downsamples to small JPEGs for a compact archive.
Preconditions / failure modes
  • Requires at least one connected browser tab (TransientSubscriberCount == 0 → error).
  • 60s default render timeout (timeout_seconds overrides).
  • Returns a structured error if the browser reports a render failure.
